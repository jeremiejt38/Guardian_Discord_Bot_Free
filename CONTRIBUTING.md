# Contributing to Guardian Discord Bot

Thank you for your interest in contributing to Guardian!

## How contributions work

Guardian is an **open-core** project. The source code is developed in a private
repository. This public repository (`Guardian_Discord_Bot_Free`) contains the
free version of the bot and is the place where you can:

- **Report bugs** via [Issues](../../issues)
- **Suggest features** via [Issues](../../issues)
- **Submit bug fixes** via [Pull Requests](../../pulls)

Accepted contributions are manually reviewed, tested, and integrated into the
private source repository. They will appear in the next public release.

## What we accept

- Bug fixes (reproducible, with a clear description)
- Performance improvements
- Documentation corrections
- Translations / i18n improvements

## What we do not accept

- New premium features (these are developed internally)
- Changes that alter the project's license or copyright notices
- Code that introduces external dependencies without prior discussion

## Submitting a bug fix

1. Open an issue first describing the bug and your proposed fix
2. Fork the repository and create a branch: `fix/description-of-bug`
3. Make your changes, keeping the code style consistent
4. Submit a pull request referencing the issue: `Fixes #N`

## Code of conduct

Be respectful. Contributions that include harassment, hate speech, or
deliberate malice will be rejected and the contributor banned.
